# Альфа Бизнес Навигатор

Telegram-бот в формате продуктового навигатора: подбор формата работы с финансами, официальный каталог продуктов, ближайшие банкоматы и отделения, карта и адреса, курс валют и админ-панель.

## Что уже есть

- главный экран с короткими действиями: рядом, подбор формата, каталог продуктов и курс валют;
- внутренний каталог продуктов в `data/content/product_catalog.json`;
- сценарии под ситуации бизнеса и подбор по отрасли;
- диагностика бизнеса за 60 секунд без раннего сбора персональных данных;
- поиск ближайших банкоматов и отделений по геопозиции;
- ручной ввод адреса как более точный fallback для случаев с VPN / proxy в Telegram;
- экран продукта с кратким описанием, документами и переходом к оформлению;
- карта и адреса через Yandex Maps / Geocoder-ready слой;
- курс валют Банка России через XML Daily;
- агентский контур с ручным одобрением по команде `/agent`;
- перед заявкой на агентский доступ бот проводит короткое собеседование, отправляет справочный PDF, запускает экзамен и только после проходного балла отправляет анкету админу;
- антифрод-скоринг лидов на доступных Telegram-сигналах;
- админ-панель с пользователями, активностью, фильтрами, quality control, агентскими заявками и CSV;
- сообщения пользователя создают новый ответ бота, а inline-кнопки обновляют текущую панель;
- тексты и дисклеймеры в `data/content/messages.json`.

## Хостинг 24/7 на Render

Для этого бота самый простой прод-вариант - `Background Worker` на Render с persistent disk, потому что:

- бот уже работает через polling и не требует webhook;
- SQLite-файл можно сохранить на диске;
- сервис будет работать независимо от вашего Mac.

В проект уже добавлен [render.yaml](/Users/timur/Documents/New project/business_start_bot/render.yaml).

Что нужно сделать:

1. Создать git-репозиторий для проекта и запушить его на GitHub, GitLab или Bitbucket.
2. В Render создать новый Blueprint или выбрать `New +` -> `Blueprint`.
3. Подключить репозиторий с этим проектом.
4. В Render заполнить секреты:
   - `TELEGRAM_TOKEN`
   - `BOT_USERNAME`
   - `OFFER_URL`
   - `YANDEX_MAPS_API_KEY`, если нужен геокодинг
   - `LEAD_WEBHOOK_URL`, `POSTBACK_URL`, `GOOGLE_SHEETS_WEBHOOK_URL`, если нужны внешние отправки
5. Дождаться первого деплоя.

Важно:

- диск монтируется в `/opt/render/project/src/data`, поэтому SQLite и экспорт сохраняются между рестартами;
- сервис запускается как worker командой `python3 run.py`;
- если потом захотите webhook вместо polling, это уже отдельная перенастройка.
